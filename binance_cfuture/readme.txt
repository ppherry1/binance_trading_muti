请务必在仔细看完邢不行关于本代码的视频讲解后再尝试运行本程序。

注意事项
1. 保证合约交易权限已经开通。需要答题。
2. 保证api已经开通合约交易权限。
3. 最大杠杆设置的大一点。比如实际下单杠杆是2倍，建议页面的最大保证金设置为5倍。以防在一些下单的瞬间保证金不足。
4. 不能同时持有某个币种的不同合约。例如不同同时持有btc的永续和季度合约。

特点
1. 币安可以一次性获取1500根k线，不用像okex那样每次获取200根然后往前推。
2. 在币安单向模式下，不用先平后开。比如目前持有1个btc多仓，策略发出做空2个btc的信号。我不需要像okex先平1个btc，再开2个btc，而是可以直接做空3个btc达到目的。
3. 币安有批量下单接口（最多5个），会使用该接口下单。
4. 本程序同时适用币本位的永续合约、交割合约。在symbol_config中设置不同的symbol即可，例如比特币永续为BTCUSD_PERP，交割为BTCUSD_210625。
3. 币安存在自动减仓机制，需要处理。

模式：
1. u模式本质还是美金本位，盈亏的是币，但是都会转换为usd来计算。
2. 币模式，即一边屯币一边做择时策略。

之后版本计划增加的内容
1. 自动发送钉钉的代码
2. 当同时持有某个币种的不同合约时报错。例如同时持有btc的永续和季度合约时，程序暂停
